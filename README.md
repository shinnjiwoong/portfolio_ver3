 ### 👀 Overview / 개요
 #### 1. 달성목표
  *이 프로젝트를 통해서 달성하고 싶었던 목표는 무엇이었는가?*
  1. 외부의 라이브러리 없이 __순수한 자바스크립트 코드__ 로만 제작한다.
  2. 디자인적으로 최대한 덜어내고, __군더더기 없이 꼭 필요한 정보로만으로__ 흥미로운 웹사이트를 구성한다.


 #### 2. HOW / 구현 방식 
 1. 전체적인 디자인 컨셉은 **화면을 4분하여 각 화면마다 다른 데이터를 보여주는 방식**으로 결정했다. 
 2. 필수적인 데이터 외에 다른 요소들은 최대한 배제하였기 때문에, 시각적인 동적 요소를 부여하기 위해서 **클릭할 때마다 화면의 색이 바뀌는 것**으로 보완하고자 했다. 
 3. 색이 바뀌는 부분에 더해서 **직접 제작한 음악**을 더해 전체적으로 지루해질 수 있는 부분을 보완하고자 했다.

 ### 👀 Improvements / 개선사항
 1. UX/UI 관련된 지식이 아직 많이 부족한 것이 가장 큰 것 같다. 주위의 사람들에게 테스팅을 부탁해보고 피드백을 받아본 결과, **여러가지 요인에 의해 웹사이트를 받아들이는 난이도에 차이를 보였다**. 따라서, **내가 구상한 컨셉을 많은 사람들이 쉽게 이해할 수 있도록 하는 방식에 대한 공부와 고민이 더 필요**하다고 생각이 든다. 특히, 앞으로는 다양한 미디어 사이즈에 디자인적으로 어떻게 대응해야할지를 초기단계에서부터 명확하게 구축해야 함을 느꼈다. 
 
 2. 여러 이미지와 영상 데이터를 빠르게 처리할 수 있는 방식 및 로딩되는 동안 지루함을 느끼지 않도록 하는 인터랙션 개발이 필요할 것으로 보인다.

### 👀 What I Learned / 이번 프로젝트로 배운것
1. **코드의 비동기적 처리**가 왜 필요한지에 대해서 실제 경험을 통해 배우고, **async/await의 사용법**에 대한 확실한 이해를 했다.
2. 다양한 사이즈의 화면에 대응하는 **미디어쿼리 개발**을 연습할 수 있었다. 
3. CSS와 자바스크립트만으로 여러가지 **인터랙션 요소**들을 만들어보는 경험이 되었다. 
4. 간단한 **웹 배포과정**을 경험할 수 있었다. 
5. **SPA (Single Page Application)** 개발의 맛을 볼 수 있었다.

### 👀 Thoughts / 고찰
이번 프로젝트는 UX적 측면에서 생각을 많이 하게 된 프로젝트였다. 개개인마다 어떤 것을 받아들이는 방식은 너무도 다르다. 하지만, 동시에 대부분의 사람들이 받아들이는 동일한 어떤 '느낌'은 존재하는 것 같다. 
개발자로서, 이러한 부분을 미리 예상하고 사용자 경험 과정을 설계하는 것이 필요하겠다. 

 - 어떤 요소에 마우스를 올리는 행위를 진행했을 때 그 요소가 어떤 방식으로던 반응을 하면 사람들은 무심코 클릭하게 된다. 이때 그 요소가 클릭에 대한 반응을 하지 않으면, 사용자의 기대감을 떨어뜨리는 계기가 되는 것 같다. 이러한 부분은 웹사이트를 주변인들에게 보여주고 피드백 받는 과정이 없었다면 빠르게 캐치하기 힘든 부분이었다. 프로젝트에 대한 피드백이 중요하다는 점과 더불어 반응하는 요소를 만들때에 사용자를 헷갈리게 하지 않도록 필요한 부분에만 넣는 과정이 필요할 것으로 보인다.
 
 - 사용자가 어떤 요소를 클릭하거나 호버하는 행위를 했을때, 본인이 지금 제대로 인터랙션을 하고 있는지에 대한 확실한 피드백을 제공해줘야한다. 이를 위해서는 사용자가 보고자 하는 내용이 명확히 있을때 (예를 들어, 어떤 아이템을 클릭해서 디테일한 정보를 얻고자 하는 경우.) 그 내용에 집중할 수 있도록 웹사이트가 환경을 제공해줘야한다. 

 - 프로젝트를 진행하는 과정에서 중간중간 덜어낼 것은 없는지를 생각해보는 시간을 가져야할 것 같다. 욕심이 많으면 무아지경으로 요소들을 추가하고 배치하게 되는 경향이 있는 것 같은데, 그걸 중간중간 찾아내지 않으면 사용자 입장에서 혼돈스러운 웹사이트를 만들기 쉽상이다. 보여주고 싶은 것들이 많을 때, 그걸 한걸음 뒤에서 전체적으로 바라보고 덜어내고 절제하는 연습도 필요할 것으로 보인다.
